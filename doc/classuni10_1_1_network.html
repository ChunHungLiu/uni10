Detailed Description

The Network class defines the tensor networks.

A Network consists of connections which is specified by labels. To construct a network, prepare a file as the following example, 

A: 1 2; 3 4
B: 3 4; 5 6
C: -1; 7 1
D: -2; 2 8
E: 7 5; -3
F: 6 8 -4
TOUT: -1 -2; -3 -4
ORDER: A B C E D F

The first column specifies the labels of the tensors. The line starting with TOUT specifies the labels of the output tensor. Labels are separated by space or comma. Incoming and outgoing labels are seperated by a semi-colon. The last line, starting from “ORDER: ” is optional. This line is used to provide a suggested order to construct the pair-wise contractions. The contraction order can be forced by putting parentheses at appropriated place as, 

ORDER: ((((A B) C) E) (D F))
Note
The TOUT: line is required. If the result is a scalar, keep the line TOUT: without any labels.

For a newtork described in the following network file,

W1: -1; 0 1 3
W2: -2; 7 10 11
U: 3 7; 4 8
Ob: 1 4; 2 5
UT: 5 8; 6 9
W1T: 0 2 6; -3
W2T: 9 10 11; -4
Rho: -3 -4; -1 -2
TOUT:
ORDER: W1 W1T W2 W2T U Ob UT Rho

Before calling launch(), the function prints out Network as

W1: i[-1] o[0, 1, 3]
W2: i[-2] o[7, 10, 11]
U: i[3, 7] o[4, 8]
Ob: i[1, 4] o[2, 5]
UT: i[5, 8] o[6, 9]
W1T: i[0, 2, 6] o[-3]
W2T: i[9, 10, 11] o[-4]
Rho: i[-3, -4] o[-1, -2]
TOUT:

i are labels for incoming bonds and o for outgoing bonds. After launch() is called, in addition to the network, it also prints out the contraction sequence, or the binary tree of the pair-wise contractions is also printed as,

*(1):
| *(70): 7, 9, -4, -2,
| | *(70): -1, 7, 9, -3,
| | | *(70): -1, 0, 7, 2, 6, 9,
| | | | W1(20): -1, 0, 1, 3,
| | | | *(20): 3, 7, 1, 2, 6, 9,
| | | | | *(20): 3, 7, 8, 1, 2, 5,
| | | | | | U(6): 3, 7, 4, 8,
| | | | | | Ob(6): 1, 4, 2, 5,
| | | | | UT(6): 5, 8, 6, 9,
| | | W1T(20): 0, 2, 6, -3,
| | Rho(924): -3, -4, -1, -2,
| *(70): -2, 7, 9, -4,
| | W2(20): -2, 7, 10, 11,
| | W2T(20): 9, 10, 11, -4,

The output shows how the network is contracted.
