Example showing how to document a function with Doxygen. 

Description of what the function does. This part may refer to the parameters of the function, like {\tt param1} or {\tt param2}. A word of code can also be inserted like {\tt this} which is equivalent to {\tt this} and can be useful to say that the function returns a {\tt void} or an {\tt int}. If you want to have more than one word in typewriter font, then just use $<$tt$>$. We can also include text verbatim, 

\footnotesize\begin{verbatim}like this\end{verbatim}
\normalsize
 Sometimes it is also convenient to include an example of usage: 

\begin{Code}\begin{verbatim} BoxStruct *out = Box_The_Function_Name(param1, param2);
 printf("something...\n");
\end{verbatim}\end{Code}

 Or, 

\begin{Code}\begin{verbatim} {.py}
 pyval = python_func(arg1, arg2)
 print pyval
\end{verbatim}\end{Code}
